## 🔴 PROBLÈMES CRITIQUES (À corriger avant publication)

### 1. **Store Rejection (Apple) - "Mot de passe oublié" non implémenté**

`src/screens/auth/ForgotPasswordScreen.tsx` est un **placeholder** :

```tsx
<Text>Page d'accueil</Text>
<Button title="Voir une annonce" onPress={() => navigation.navigate("AnnouncementDetail")} />
```

**Impact** : **Rejet Apple probable** pour fonctionnalité incomplète

**Solution** : Implémenter via `supabase.auth.resetPasswordForEmail()`

---

### 2. **Sécurité - UserId manipulable côté client**

Plusieurs endpoints API passent le `userId` en paramètre au lieu de le récupérer depuis la session Supabase :

| Fichier                                 | Impact                                         |
| --------------------------------------- | ---------------------------------------------- |
| `src/api/announcement/apply.service.ts` | Postuler au nom d'un autre utilisateur         |
| `src/api/announcement/getFavorite.ts`   | Manipulation des favoris d'autres utilisateurs |

**Impact** : Faille de sécurité permettant l'usurpation d'identité (si RLS non configuré)

**Solution** : Récupérer systématiquement `userId` via `supabase.auth.getSession()` ou s'assurer que les RLS Supabase bloquent ces accès

---

### 3. **Performance - Pagination manquante sur les messages**

`src/screens/chat/ChatScreen.tsx` charge TOUS les messages d'une conversation sans pagination.

**Impact** : Freeze de l'application sur les longues conversations

**Solution** : Ajouter `.range(0, 50)` + scroll infini inversé

---

### 4. **Performance - Refetch complet sur nouveau message**

Dans `src/screens/chat/ChatScreen.tsx`, le listener realtime appelle `refetch()` à chaque nouveau message au lieu d'ajouter le message au state.

**Impact** : Performance dégradée, requêtes inutiles

**Solution** : Ajouter le nouveau message directement au state local
