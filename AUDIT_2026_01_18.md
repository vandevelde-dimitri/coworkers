## 🔴 PROBLÈMES CRITIQUES (À corriger avant publication)

### 1. **Sécurité - UserId manipulable côté client**

Plusieurs endpoints API passent le `userId` en paramètre au lieu de le récupérer depuis la session Supabase :

| Fichier                                            | Impact                                                  |
| -------------------------------------------------- | ------------------------------------------------------- |
| `src/api/messaging/addUserConversation.ts`         | Injection dans n'importe quelle conversation            |
| `src/api/announcement/apply.service.ts`            | Postuler au nom d'un autre utilisateur                  |
| `src/api/announcement/getFavorite.ts`              | Manipulation des favoris d'autres utilisateurs          |
| `src/api/messaging/getUserConversationsPreview.ts` | Lecture des conversations de n'importe quel utilisateur |

**Impact** : Faille de sécurité majeure permettant l'usurpation d'identité

**Solution** : Récupérer systématiquement `userId` via `supabase.auth.getSession()` côté serveur

---

### 2. **Store Rejection (Apple) - Suppression de compte non fonctionnelle**

Dans `src/screens/profile/SettingsScreen.tsx`, le bouton "Supprimer mon compte" affiche un toast de succès mais **n'effectue aucune action réelle** :

```typescript
onConfirm={() => {
    setOpenDestroy(false);
    showToast("success", "Suppression de compte réussie"); // ❌ Pas de vraie suppression !
}}
```

**Impact** : **Rejet Apple garanti** - Apple exige depuis 2022 une vraie fonctionnalité de suppression de compte

**Solution** : Implémenter la suppression via `supabase.auth.admin.deleteUser()` (nécessite une fonction Edge)

---

### 3. **Store Rejection (Apple) - "Mot de passe oublié" non implémenté**

`src/screens/auth/ForgotPasswordScreen.tsx` est un **placeholder** :

```tsx
<Text>Page d'accueil</Text>
<Button title="Voir une annonce" onPress={() => navigation.navigate("AnnouncementDetail")} />
```

**Impact** : **Rejet Apple probable** pour fonctionnalité incomplète

**Solution** : Implémenter via `supabase.auth.resetPasswordForEmail()`

---

### 4. **Store Rejection - Permissions microphone contradictoires**

Dans `app.json` :

- `NSMicrophoneUsageDescription`: "Coworkers n'utilise pas le microphone"
- Plugin `expo-camera` : `"recordAudioAndroid": true`

**Impact** : Apple rejettera pour incohérence permissions/justification

**Solution** :

```json
[
    "expo-camera",
    {
        "cameraPermission": "...",
        "recordAudioAndroid": false
    }
]
// Supprimer microphonePermission
```

---

### 5. **Store Rejection - Identifiants d'application manquants**

`app.json` manque :

- iOS : `bundleIdentifier`
- Android : `package`
- Numéros de build : `buildNumber` / `versionCode`

**Solution** :

```json
"ios": {
    "bundleIdentifier": "com.votreentreprise.coworkers",
    "buildNumber": "1"
},
"android": {
    "package": "com.votreentreprise.coworkers",
    "versionCode": 1
}
```

## 🟠 AMÉLIORATIONS RECOMMANDÉES

### Sécurité

| Fichier                                       | Problème                                              | Recommandation                        |
| --------------------------------------------- | ----------------------------------------------------- | ------------------------------------- |
| Multiples fichiers API                        | `console.log` exposant des données                    | Supprimer ou conditionner à `__DEV__` |
| `src/contexts/authContext.tsx`                | `console.error("refreshSession error")` en production | Utiliser un logger conditionnel       |
| `src/api/messaging/getConversationMessage.ts` | Pas de vérification d'appartenance à la conversation  | Valider via RLS Supabase              |

### Stabilité

| Fichier                                      | Problème                                    | Recommandation                                 |
| -------------------------------------------- | ------------------------------------------- | ---------------------------------------------- |
| `src/screens/chat/ConversationsScreen.tsx`   | `if (isLoading) return null;` → écran blanc | Afficher un `<ActivityIndicator>`              |
| `src/screens/chat/ChatScreen.tsx`            | `handleSend()` sans try/catch               | Ajouter gestion d'erreur + toast               |
| `src/screens/auth/SignUpScreen.tsx`          | Pas d'état loading sur le bouton            | Ajouter `isSubmitting` pour éviter double-clic |
| `src/screens/auth/CompleteProfileScreen.tsx` | Multiples `await` sans try/catch global     | Entourer de try/catch                          |
| Toutes les mutations React Query             | Pas de handler `onError`                    | Ajouter toast d'erreur utilisateur             |

### Performance

| Fichier                                      | Problème                             | Recommandation                            |
| -------------------------------------------- | ------------------------------------ | ----------------------------------------- |
| `src/screens/chat/ChatScreen.tsx`            | Pas de pagination des messages       | Ajouter `.range(0, 50)` + infinite scroll |
| `src/screens/chat/ChatScreen.tsx`            | `refetch()` à chaque nouveau message | Ajouter le message au state directement   |
| `src/api/getAllFc.ts`                        | `select("*")` sans pagination        | Sélectionner colonnes spécifiques         |
| Multiples fichiers API                       | `select("*")` → données inutiles     | Lister les colonnes nécessaires           |
| `src/hooks/announcement/useAnnouncements.ts` | QueryKey sans pagination             | Ajouter `page, pageSize` à la queryKey    |

### React Native Best Practices

| Fichier                                                               | Problème                            | Recommandation                      |
| --------------------------------------------------------------------- | ----------------------------------- | ----------------------------------- |
| Contextes (`authContext`, `messageContext`, `notificationContext`)    | Value du Provider non mémorisée     | Wrapper avec `useMemo`              |
| Composants liste (`ProfileCard`, `ConversationItem`, `MessageBubble`) | Pas de `React.memo()`               | Ajouter pour optimiser FlatList     |
| `src/navigation/AppTabs.tsx`                                          | Styles inline dans options          | Extraire dans `StyleSheet.create()` |
| `src/navigation/ProfileStack.tsx`                                     | Import `useRequireAuth` non utilisé | Supprimer                           |
